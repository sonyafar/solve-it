Golden-section Search is a variation of Bisection method and starts with two initial guesses say x0 and x1 such that x0 and x1 brackets the root i.e. f(x0)·f(x1)< 0.

Unlike the Bisection method, the interval is not divided in half, but by the value N≈1.618. If x0 and x1 are two guesses then we compute a new approximated value as:

{{'$$x_2 = x_0 + \\frac{x_1 - x_0}{1.618}$$'}}

which then is used as one of the two bracketing values in the next step of the iteration.

Here we have following three different cases:

  1. If f(x2)=0 then the root is x2.
  2. If f(x0)·f(x2)<0 then root lies between x0 and x2.
  3. If f(x0)·f(x2)>0 then root lies between x1 and x2.

So that after each iteration we redefine bracketing values and repeat the calculation process until we find the root within desired accuracy.
